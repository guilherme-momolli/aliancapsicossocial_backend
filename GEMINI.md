# AI Agent Guidelines - Aliança Psicossocial

Este arquivo serve como o ponto de entrada principal para agentes de IA operando neste repositório. Ele define as diretrizes fundamentais para garantir consistência, qualidade e eficiência na comunicação e geração de código.

## Objetivo
Garantir que todos os agentes de IA sigam os mesmos padrões de excelência técnica (Clean Code/Architecture) e diretrizes de escrita (Markdown eficiente).

## Documentação de Referência para IA

Para diretrizes específicas, consulte os arquivos abaixo:

### Escrita e Markdown
* **[Padrões de Markdown](docs/ai-guidelines/markdown-standards.md):** Regras para geração de texto, como evitar redundância e manter a concisão.

### Arquitetura e Código
* **[Clean Standards](docs/ai-guidelines/clean-standards.md):** Aplicação de Clean Code e Clean Architecture em todas as entregas.
* **[Boas Práticas do Projeto](docs/boas-praticas.md):** Índice de padrões técnicos (Java, Spring, Git, etc.).

## Mandatos Principais
1. **Concisão:** Nunca repita informações já fornecidas no contexto ou em respostas anteriores.
2. **Qualidade Técnica:** Todo código gerado deve seguir rigorosamente as normas de Clean Code e a arquitetura definida no projeto.
3. **Validação:** Sempre valide alterações com testes e comandos de linting/build disponíveis.

---
*Este arquivo deve ser lido por todos os agentes no início de cada sessão.*
